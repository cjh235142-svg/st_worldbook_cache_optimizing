# 交互流程

## 流程 1：打开世界书文件

```
用户操作                           GUI 响应
─────────                        ─────────
1a. 文件→打开 (Ctrl+O)            弹出 QFileDialog → 选择 .json
                                   ↓
1b. 拖拽 .json 到窗口              WorldBookTree.dragEnterEvent → 接受
                                   WorldBookTree.dropEvent → 发射 _on_file_dropped(path)
                                   MainWindow 连接该信号 → 调用 load
                                   ↓
                                  加载 JSON → 校验 entries 字典
                                   ↓
                                  WorldBookModel 构建数据
                                   ↓
                                  左树 WorldBookTree 构建根节点+有数据的层级1-2
                                  （跳过无条目的 position 组）
                                   ↓
                                  更新 StatsBar.set_stats(总/静/动/混)

2. 展开/折叠树节点                 点击 ▶/▼ 切换
                                   ↓
                                  展开层级2 → 懒加载层级3 (条目)
                                  展开层级3 → 懒加载层级4 (字段详情)
                                  折叠层级3 → 销毁 QTextBrowser (释放内存)

3. 点击折叠工具栏按钮               collapse_to_level(1|2|3)
```

## 流程 2：条目拆分

```
用户操作                           GUI 响应
─────────                        ─────────
1. 点击工具栏「运行分析」            调用 subprocess:
                                   python -m scripts.analyze_entries
                                     -i <原文件> -o <tmp/_analysis.json>
                                   ↓
                                   解析 _analysis.json
                                   ↓
                                   层级3中 suggested_split=true 的条目
                                   出现 CheckBox (ROLE_CHECKABLE)
                                   ↓
                                   状态栏提示「分析完成」

2. 展开包含可拆分条目的节点          条目左侧显示 ☐ 复选框

3. 勾选要拆分的条目                  树条目 checkState 更新

4. 点击「预览拆分」                  调用 subprocess:
                                    python -m scripts.split_entries
                                      -i <原文件> -a <.wb_optimize/_analysis.json>
                                      -o <.wb_optimize/_split.json>
                                    ↓
                                    解析 _split.json（拆分全部 suggested_split=true 的条目）
                                    更新树条目右栏 ROLE_RIGHT_TEXT
                                    （无论是否勾选，预览时全部拆分并显示结果）
                                    ↓
                                    右面板切换到预览模式
                                    (只显示控制按钮, 结果在树中查看)

5. 查看拆分结果                      展开树节点，右栏显示映射：
                                    已拆: → 静态子条目 / 动态子条目
                                    未拆: → 不变 (右栏灰底)

6. 点击右侧面板「确认拆分」           保存当前展开状态 uids = save_expanded_state()
                                    ↓
                                    收集所有勾选的条目 UID（checked_uids）
                                    ↓
                                    解析 _split.json，**只将 checked_uids 中的条目拆分结果写回模型**
                                    未勾选的 suggested_split=true 条目保持原样
                                    ↓
                                    更新 StatsBar，重建树
                                    ↓
                                    恢复展开状态 restore_expanded_state(uids)
                                    ↓
                                    提示「拆分完成」

7. 点击「取消」                      清空右栏，恢复原状
```

## 流程 3：优化对比

### 3.1 条目级 diff

```
用户操作                           GUI 响应
─────────                        ─────────
1. 右面板切换到「对比」模式           加载对比面板
                                    自动调用 check_for_optimized(source_path)
                                    扫描 {文件目录}/.wb_optimize/{文件名}_merged.json
                                    若不存在，显示提示
                                    「未检测到优化结果，请先执行优化」
                                    + [浏览选择优化后文件...] 按钮

2. 选择子模式「条目级 diff」          默认选中（需优化文件就绪）

3. 确保优化后文件路径已设置           约定式自动检测 `.wb_optimize/` 目录
                                    或用户通过浏览按钮手动选择

4. （自动触发「加载对比数据」）        调用 subprocess:
                                    python -m scripts.tools.tool_diff
                                      --original <原> --optimized <优化后>
                                      -f json
                                    ↓
                                    解析 JSON → 更新树条目的右栏
                                    - 保持: → order:N→N'
                                    - 拆分: → 静止组X, 动uid=N
                                    - 合并: → 合并入 组X
                                    - 新建: → 管线自动生成

5. 展开树节点查看                    右栏着色区分类型
```

### 3.2 全文级 diff

```
用户操作                           GUI 响应
─────────                        ─────────
1. 切换子模式为「全文级 diff」        FulltextDiffView 显示
                                   ↓
2. 点击「生成 diff」                 将两文件内容排序拼接
                                   ↓
                                   difflib.HtmlDiff.make_file()
                                   生成双栏对比 HTML
                                   ↓
                                   渲染到单个 QTextBrowser
                                   ↓
3. 滚动查看                         浏览器的原生滚动
```

## 流程 4：测试验证

```
用户操作                           GUI 响应
─────────                        ─────────
1. 右面板切换到「测试」模式           TestPanel 显示
                                   ↓
2. 选择测试文件：
   ● 当前世界书 (默认)
   ○ 选择其他文件 [浏览...]

3. (可选) 勾选「严格模式 (--strict)」   默认关闭，仅对优化后世界书启用

4. 点击「运行验证」                   subprocess:
                                    python -m scripts.tools.tool_validate
                                      -i <文件> [--strict]
                                      --output-errors <tmp/_errors.json>
                                    ↓
                                    解析错误 JSON → 填充表格
                                    ↓
5. 点击错误表格某行                  下方面板更新：
                                    - content 全文（行号标注）
                                    - 字段列表
```

## 流程 5：一键优化与覆盖

```
用户操作                           GUI 响应
─────────                        ─────────
1. 右面板切换到「优化」模式           OptimizePanel 显示
                                    ↓
                                    显示当前文件路径
                                    包裹名输入框默认 "补充内容"

2. (可选) 修改包裹名                 输入 <包裹名>
                                    实时验证：不能为空/含空格/引号/特殊字符
                                    无效时显示红色错误标签，禁用 [运行管线] 按钮

3. 点击「运行管线 (一键优化)」        [运行管线] 禁用（防止并发）
                                    [取消优化] 显示
                                    创建临时目录 .wb_optimize/
                                    ↓
                                    启动 PipelineWorker(input_path, .wb_optimize/, wrapper_name)

   step_done(1, "备份原文件")         备份: shutil.copy2(原文件, {原文件目录}/backup/)
                                    备份路径: backup/{文件名}.backup_{YYYYMMDD_HHMMSS}.json
                                    更新进度条 1/5 + 步骤列表 ✓

   step_done(2, "分析条目")          python -m scripts.analyze_entries
                                      -i <原文件> -o <.wb_optimize/_analysis.json>
                                    更新进度条 2/5

   step_done(3, "拆分条目")          python -m scripts.split_entries
                                      -i <原文件> -a <.wb_optimize/_analysis.json>
                                      -o <.wb_optimize/_split.json>
                                      -w <包裹名>
                                    更新进度条 3/5

   step_done(4, "重排序")            python -m scripts.reorder_entries
                                      -i <.wb_optimize/_split.json>
                                      -o <.wb_optimize/_reordered.json>
                                      -w <包裹名>
                                    更新进度条 4/5

   step_done(5, "合并静态条目")       python -m scripts.merge_entries
                                      -i <.wb_optimize/_reordered.json>
                                      -o <.wb_optimize/_merged.json>
                                    更新进度条 5/5
                                    ↓
                                    pipeline_finished emit
                                    读取 .wb_optimize/_merged.json
                                    更新优化摘要区域
                                    启用 [覆盖原文件] [另存为] 按钮
                                    ↓
                                    [运行管线] 恢复可用，[取消优化] 隐藏

   — 若用户点击 [取消优化] —
                                    PipelineWorker.cancel() → 当前步骤完成后立即停止
                                    pipeline_cancelled emit → 恢复按钮状态

   — 若某步失败 —
                                    step_error(index, type, msg)
                                    弹错误对话框，保留临时产物供调试

4a. 点击「覆盖原文件」               校验 merged.json 合法性（含 entries 字典）
                                    ↓
                                    保存当前展开状态 uids = save_expanded_state()
                                    ↓
                                    合法 → 写回原路径
                                    ↓
                                    提示「优化完成，备份至 backup/」
                                    ↓
                                    重建 WorldBookModel 和树
                                    ↓
                                    恢复展开状态 restore_expanded_state(uids)

4b. 点击「另存为...」               QFileDialog → 保存 merged.json

4c. 点击「取消」                     不做写入
```

## 异常流程

| 场景 | 处理 |
|------|------|
| 脚本执行超时 (>120s) | 弹错误对话框，保留部分临时产物 |
| 脚本返回非零退出码 | 显示 stderr，建议用户检查文件 |
| 备份目录创建失败 | 提示权限不足 |
| 覆盖校验失败 | 拒绝覆盖，提示保留临时目录 |
| 文件被外部修改 | 校验修改时间，提示重新加载 |
| 打开无效 JSON | 弹错误对话框 |
| 用户取消优化 | 当前步骤正常完成后停止；已生成的临时产物保留供后续手动使用 |
| 包裹名格式无效 | 禁用 [运行管线] 按钮，QLineEdit 下方显示红色错误标签 |
