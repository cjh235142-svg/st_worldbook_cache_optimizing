# 需求规格

## 需求 1：世界书文件浏览

### 1.1 打开文件

- 用户通过「文件 → 打开」(Ctrl+O) 或拖拽 `.json` 文件到窗口打开
- 加载后在左侧树形面板中展示条目结构

### 1.2 树形展示（4 级折叠层级）

树形结构以原始世界书的组织方式为准，分为 4 级可折叠层级：

```
层级 0: 世界书根节点 (文件名)
  ├─ 层级 1: 按 position 分组
  │    ├─ 层级 2: 按条目类别分组 (静态 / 动态 / 混合)
  │    │    ├─ 层级 3: 具体条目 (comment + uid + 双栏映射)
  │    │    │    ├─ 层级 4: 字段详情 (展开后显示 content 全文、key、order 等)
```

- 层级 3 的条目在双栏 Delegate 中渲染：左栏 = 条目摘要 + 可选 □ 复选框，右栏 = 映射信息
- 层级 4 在用户展开条目节点时懒加载：
  - `content` 节点内嵌 `QTextBrowser` 显示全文
  - 其余字段（key, order, position 等）单行显示
- 条目图标：🔵 蓝灯 (constant=true) / 🟢 绿灯 / 📄 动态

### 1.3 折叠控制

顶部工具栏提供 3 个折叠层级快捷键：

| 按钮 | 行为 |
|------|------|
| 折叠到 position | 展开到层级 1，折叠 2-4 |
| 折叠到类别 | 展开到层级 2，折叠 3-4 |
| 折叠到条目 | 展开到层级 3，折叠 4 |

折叠状态同步影响右侧面板（共用树，天然同步）。

### 1.4 统计摘要

打开文件后自动计算并在状态栏显示：总条目数 / 静态条目数 / 动态条目数 / 混合条目数。

## 需求 2：条目拆分

### 2.1 触发拆分

- 用户点击「运行分析」→ 调用 `analyze_entries.py`
- 分析完成后，层级 3 中 `suggested_split=true` 的条目左侧出现 CheckBox
- 用户勾选条目，点击「预览拆分」

### 2.2 拆分预览

- 右侧面板切换到预览模式（只显示 [确认拆分] [取消] 按钮）
- 树的右栏更新为拆分映射信息：
  - 已拆分条目：右栏显示 `→ 静态子条目 / 动态子条目`
  - 未拆分条目：右栏显示 `不变`

### 2.3 确认/取消

- 「确认拆分」：只将用户勾选条目的拆分结果写回模型（未勾选的 `suggested_split=true` 条目保持原样），更新树
- 「取消」：清空右栏映射，恢复原状

### 2.4 边界情况

| 场景 | 行为 |
|------|------|
| 无可拆分条目 | 提示，预览按钮禁用 |
| outlet 条目 (pos=7) | 不可选 |

## 需求 3：优化前后对比

### 3.1 两种子模式

| 模式 | 视图 |
|------|------|
| 条目级 diff | 共用树的右栏显示映射关系（合并/拆分/保持） |
| 全文级 diff | 单 QTextBrowser 渲染 difflib 双栏 HTML |

### 3.2 条目级 diff

- 调用 `tool_diff.py -f json`，解析后更新树的右栏
- 映射类型：保持、拆分、合并、新建（管线自动生成）

### 3.3 全文级 diff

- 将两文件 entries content 按管线排序拼接
- `difflib.HtmlDiff.make_file()` 生成完整双栏对比 HTML
- 渲染在单个 QTextBrowser 中，自适应滚动

## 需求 4：测试验证

### 4.1 运行测试

- 选择文件（当前 / 其他）和严格模式复选框（`--strict`，默认关闭，仅对优化后的世界书启用）
- 点击「运行验证」，后台调用 `tool_validate.py [--strict] --output-errors <tmp.json>`

### 4.2 错误展示

- **错误表格**（上方）：级别 / 代码 / UID / Comment / 消息
- **条目上下文**（下方）：点击表格行后显示该条目 content 全文（行号标注）和字段

## 需求 5：一键优化与覆盖

### 5.1 分步优化

GUI 分 5 步调用管线（非 `run_pipeline` 单次调用），以实现进度追踪：

| 步骤 | 操作 |
|------|------|
| 1/5 | 备份原文件到 `backup/` |
| 2/5 | `python -m scripts.analyze_entries` |
| 3/5 | `python -m scripts.split_entries`（可附加 `-w <包裹名>`） |
| 4/5 | `python -m scripts.reorder_entries`（可附加 `-w <包裹名>`） |
| 5/5 | `python -m scripts.merge_entries` |

### 5.2 包裹名

用户可在优化面板中输入补充包裹的名称（默认：`补充内容`），通过 `-w` 参数传递给 split 和 reorder 步骤。

**包裹名验证规则**：
- 不能为空
- 不能含空格、引号（单/双）、反斜杠、尖括号（`<` `>`）、管道符（`|`）
- 无效时 QLineEdit 下方显示红色错误标签，禁用 [运行管线] 按钮

### 5.3 覆盖与备份

- 备份存储在 `{原文件所在目录}/backup/`，文件名：`{文件名}.backup_{YYYYMMDD_HHMMSS}.json`
- 覆盖前校验 `_merged.json` 合法性（含 `entries` 字典）
- 校验失败则拒绝覆盖，提示保留临时产物
