# GUI 应用程序概述

## 1. 项目背景

现有世界书优化管线（`scripts/`）提供了完整的 CLI 流程（分析→拆分→重排序→合并），用于最大化 SillyTavern World Info 在 DeepSeek API 下的缓存命中率。CLI 方式对非技术用户不友好，且缺少可视化交互。

## 2. 项目目标

构建 PyQt6 桌面 GUI，封装管线脚本，提供 5 项核心能力：

| # | 功能 | 描述 |
|---|------|------|
| 1 | 世界书浏览 | 打开 JSON，4 级可折叠树展示全部条目，展开查看全文 |
| 2 | 条目拆分 | 勾选 → 预览（树右栏）→ 确认 |
| 3 | 优化对比 | 条目级 diff（树的右栏）+ 全文级 diff（双栏 HTML）|
| 4 | 测试验证 | tool_validate，错误表格 + 点击查看上下文 |
| 5 | 一键优化 | 分 5 步 + 进度追踪 + 备份覆盖 |

## 3. 技术选型

| 层面 | 选择 | 理由 |
|------|------|------|
| GUI 框架 | PyQt6 | 树形/表格/富文本原生支持，跨平台 |
| 脚本调用 | subprocess | 不修改现有代码，隔离性好 |
| 文本 diff | difflib.HtmlDiff.make_file() | 标准库，生成完整双栏 HTML |
| 条目 diff | tool_diff CLI | 复用已有拆分/合并追踪 |
| 验证工具 | tool_validate CLI | 复用已有 XML/EJS/宏检查 |

## 4. 主布局

左右面板布局，共用树结构：

```
┌──────────────────────────────────────────────────────────────────┐
│  ToolBar: [打开] [运行分析] [对比] [测试] [优化]                 │
├──────────────────────┬───────────────────────────────────────────┤
│  左面板 (共用树)      │  右面板 (按模式切换)                      │
│                      │                                           │
│  4 级可折叠树形       │  [预览] [对比] [测试] [优化]  ← 模式按钮  │
│                      │                                           │
│  ▸ pos=0             │  ┌─ 预览 ────────────────────────────┐  │
│    ▸ 静态             │  │ [确认拆分] [取消]                  │  │
│      ▸ 条目A uid=1   │  │ (结果在树的右栏中查看)             │  │

│                      │  │ [双栏 HTML diff]                  │  │
│                      │  └────────────────────────────────────┘  │
│                      │                                           │
│                      │  ┌─ 测试 ────────────────────────────┐  │
│                      │  │ 错误表格 + 上下文面板              │  │
```

## 5. 关键设计原则

1. **不侵入现有管线**：GUI 通过 subprocess 调用 CLI，不修改 `scripts/`
2. **以左侧为准**：树结构以原始世界书为准，右侧只显示映射关系
3. **懒加载**：树节点按需创建（层级 1-2 初始化，层级 3-4 展开时加载）
4. **共用树**：折叠/展开左右天然同步，因为同一棵树
5. **安全第一**：优化前自动备份到 `{文件目录}/backup/`，备份名 `{文件名}.backup_{YYYYMMDD_HHMMSS}.json`
6. **分步进度**：优化模式下分别调用 4 个脚本，每步完成后更新进度
7. **状态保持**：拆分/优化后重建树时保存并恢复展开状态

## 6. 文件位置

- GUI 源码：`app/main.py`（单入口，多模块分层）
- 设计文档：`docs/GUI/*.md`
