## 工具 3：`tool_prompt.py` — 生成完整参考提示词

```
python tools/tool_prompt.py \
    --input <世界书.json> \
    [--system-prompt <system.txt>] \
    [--character <角色卡.json>] \
    [--persona <用户.json>] \
    [--scenario <场景.txt>] \
    [--chat-history <酒馆聊天导出.json>] \
    [--output <out.txt>] \
    [--no-comments]
```

| 参数 | 说明 |
|---|---|
| `--input` | 世界书 JSON（必需） |
| `--system-prompt` | 系统提示词文件（可选，缺省用占位符） |
| `--character` | 角色卡 JSON 路径（可选，提取 description/personality/scenario） |
| `--persona` | 用户 Persona JSON 路径（可选） |
| `--scenario` | 场景描述文件（可选，覆盖角色卡中的 scenario） |
| `--chat-history` | 酒馆导出的聊天 JSON 路径（可选，用于模拟 atDepth 注入位置） |
| `--output` | 输出到文件（默认 stdout） |
| `--no-comments` | 仅输出纯内容，不含 HTML 注释 |

### 模拟的提示词构建流程

与酒馆 Prompt Manager 默认顺序一致（见前置知识 2.3 节）：

```
1. System Prompt (Main Prompt)
2. World Info (before): pos=0 条目
3. Persona Description
4. Character Description
5. Character Personality
6. Scenario
7. World Info (after): pos=1 条目
8. Chat Examples: pos=5/6 条目
9. Chat History:
   ├─ msg_1
   ├─ ... (depth=N 处注入 pos=4 条目)
   ├─ msg_n-1 (depth=1 处注入 pos=4 条目)
   └─ msg_n (depth=0 处注入 pos=4 条目，最后一条之后)
10. Post-History Instructions (AN: pos=2/3 条目)
```

### chat-history 解析

```python
def parse_chat_history(path: str) -> list[dict]:
    """
    加载酒馆聊天 JSON，提取消息列表。
    兼容多种导出格式：
    - data["chat"]  (酒馆标准导出)
    - data["messages"]
    每条消息提取 role (user/assistant/system) 和 content。
    """
    with open(path, "r", encoding="utf-8") as f:
        data = json.load(f)

    messages = data.get("chat", data.get("messages", []))

    return [
        {
            "role": m.get("role", "user"),
            "content": m.get("content", ""),
        }
        for m in messages
    ]
```

### atDepth 注入模拟

按 depth 从大到小将 `pos=4` 条目注入消息序列：

```
depth=99  → 注入到第 0 条消息之前
depth=10  → 注入到第 1 条消息之前
...
depth=1   → 注入到倒数第 2 条消息之前
depth=0   → 注入到最后一条消息之后
```

## 工具 4：`tool_validate.py` — 合法性检查

```
python tools/tool_validate.py --input <世界书.json> [--output-errors <out.json>] [--strict]
```

| 参数 | 说明 |
|---|---|
| `--input` | 世界书 JSON 路径 |
| `--output-errors` | 错误列表输出为 JSON（机器可读） |
| `--strict` | 严格模式，额外检查建议性规则 |

### 检查项分类

#### A. 结构合法性（🔴 错误，必须修复）

| # | 检查项 | 范围 | 说明 |
|---|---|---|---|
| A1 | JSON 解析失败 | 文件级 | 文件不是合法 JSON |
| A2 | entries 不是字典 | 文件级 | 顶层结构错误 |
| A3 | uid 与 entries key 不一致 | 条目级 | `entry["uid"] != int(entries_key)` |
| A4 | 必需字段缺失 | 条目级 | content 字段不存在 |

#### B. EJS 完整性（🔴 错误）

| # | 检查项 | 范围 | 说明 |
|---|---|---|---|
| B1 | `<%` 无对应 `%>` | 条目内 | EJS 代码块未闭合 |
| B2 | EJS brace 不配对 | 条目内 | `<% if { %>` 无对应 `<% } %>`，追踪复合块 brace 净值是否为 0 |

#### C. XML 标签配对（🔴 错误）

| # | 检查项 | 范围 | 说明 |
|---|---|---|---|
| C1 | 开标签无闭合 | 条目内 | `<角色>` 无 `</角色>` |
| C2 | 闭标签无开头 | 条目内 | `</角色>` 无 `<角色>` |
| C3 | 标签交叉 | 条目内 | `<A><B></A></B>` |
| C4 | 跨条目开标签无闭合 | **全局（所有 position/depth 组）** | 将所有条目 content 按排序拼接，检测 XML 标签配对 |
| C5 | 跨条目闭标签无开头 | **全局（所有 position/depth 组）** | 同上 |

> C4/C5 不限制 scope 为同组。例如 `pos=0 depth=4` 的 `<暗部>` 应由 `pos=0 depth=99` 的 `</暗部>` 闭合——全局配对检测能发现此类跨组标签。

#### D. Markdown 标题跨条目连贯性（🟡 警告）

| # | 检查项 | 范围 | 说明 |
|---|---|---|---|
| D1 | H1 无后续内容 | 条目级 | 条目仅含 `# 标题` 无正文（可能是分界条目标记） |
| D2 | H1 开闭不配对 | **全局（所有 position/depth 组）** | `# XXX 开始`（条目A）无对应 `# XXX 结束`（任意条目） |

> 仅检查 comment 中包含 `开始`/`start`/`begin` 或 `结束`/`end`/`close` 的条目（他语言等价亦可扩展）。不对普通标题做强制配对。

#### E. 宏完整性（🟡 警告）

| # | 检查项 | 范围 | 说明 |
|---|---|---|---|
| E1 | `{{` 无对应 `}}` | 条目内 | 宏未闭合 |
| E2 | EJS 输出中嵌套宏 | 条目内 | `<%= {{...}} %>`（可疑但可能合法） |

#### F. 缓存优化规则检查（🟡 警告，仅 `--strict`）

| # | 检查项 | 说明 |
|---|---|---|
| F1 | `constant=true` 但 `key` 非空 | 蓝灯无需关键词 |
| F2 | `useProbability=true` 且 `probability < 100` 的静态条目 | 缓存优化目标要求必定激活 |
| F3 | 动态条目 `pos != 4` | 后续脚本会改到 pos=4，提醒用户 |
| F4 | `depth >= 10` 但 `pos != 4` | depth 仅在 atDepth 下有意义 |
| F5 | `content` 为空或仅空白 | 无实际内容 |
| F6 | order 值重复 | 同 `(pos, depth)` 组内 order 重复可能导致不稳定排序 |

### 输出格式

```
══════════════════════════════════════════════════════
  世界书合法性检查: 学园都市0313.json
══════════════════════════════════════════════════════

🔴 错误 (4):
  [A3] uid=99: uid 与 entries key 不一致 (uid=99, key="100")
  [B2] uid=23 "事件：科学": EJS brace 不配对 (行 5-12)
  [C4] 全局: XML 标签 <暗部> 跨条目未闭合
        开标签: uid=15 "暗部" (pos=0 depth=4)
        无对应 </暗部> 在任意条目中
```

## 工具 5：`tool_diff.py` — 排序后对比

```
python tools/tool_diff.py --original <原.json> --optimized <优化后.json>
    [--format unified|json] [--output <file>] [--max-order-changes N]
```

| 参数 | 说明 |
|---|---|
| `--original` | 原始世界书 JSON |
| `--optimized` | 管线优化后的世界书 JSON（通常为 merged 产物） |
| `--format` | `unified`（默认）或 `json` |
| `--output` | 输出到文件（默认 stdout） |
| `--max-order-changes` | order 变更显示上限，`0` = 无限制（默认），`N > 0` = 只显示前 N 条 |

### `unified` 格式

```
═══════════════════════════════════════════════════
  Diff: 学园都市0313.json → 学园都市0313_merged.json
═══════════════════════════════════════════════════

--- Entry Count
  原始: 163
  优化: 120
  Δ: -43 (8 组合并缩减, 2 条拆分增加)

--- Order Changes (163 条重排)
  uid=5  "纯静态条目"           100 → 0    Δ: -100
  uid=8  "核心01 [split-static]" 50 → 1    Δ: -49
  uid=12 "核心01 [split-dynamic]" 30 → 21  Δ: -9
  ... (--max-order-changes 限制显示数量，0=全部)

--- Field Changes: uid=5 ---
  constant:        false → true
  useProbability:  true  → false
  cooldown:        0     → 0 (unchanged)

--- Merged Entries (8 groups) ---
  合并:0-2    ← uid=3 + uid=7 + uid=12
    pos=4 depth=0 constant=true
  合并:3-5    ← uid=15 + uid=18
    pos=0 depth=4 constant=true
  ...

--- Split Entries (2 entries) ---
  uid=20 "事件：背景"
    → [split-static] uid=20
    → [split-dynamic] uid=21

--- Removed ---
  originalData: (removed)

--- Summary ---
  Entries:     163 → 120 (-43)
  Static:      100 → 8   (合并后)
  Dynamic:     63  → 63  (不变)
  Order range: 1..500 → 0..119
═══════════════════════════════════════════════════
```

### `json` 格式

```json

  "original": "学园都市0313.json",
  "optimized": "学园都市0313_merged.json",
  "entry_count": {"before": 163, "after": 120},

    {"uid": 5, "comment": "纯静态条目", "old": 100, "new": 0}
  ],
  "field_changes": [
    {"uid": 5, "field": "constant", "old": false, "new": true}
  ],
  "merged_groups": [
    {"comment": "合并:0-2", "sources": [3, 7, 12]}
  ],
  "split_entries": [
    {"uid": 20, "children": ["事件：背景 [split-static]", "事件：背景 [split-dynamic]"]}
  ],
  "removed": ["originalData"]
}
```
