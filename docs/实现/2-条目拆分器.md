# 脚本 2 设计文档：条目拆分器 `split_entries.py`

---

## 1. 输入参数

```bash
python split_entries.py --world-book <世界书.json> --analysis <分析结果.json> [--output <输出路径.json>]
```

| 参数 | 必需 | 默认值 | 说明 |
|---|---|---|---|
| `--world-book` | 是 | — | 原始世界书 JSON 路径（content 完整，只读） |
| `--analysis` | 是 | — | 脚本 1 输出的 `_analysis.json` |
| `--output` | 否 | `{世界书原名}_split.json` | 拆分后的世界书输出路径 |

---

## 2. 输入/输出关系

```
输入A: 学园都市0313.json          ← 原始世界书（只读，不修改）
输入B: 学园都市0313_analysis.json  ← 脚本1输出（含 uid → 拆分判定 + 边界信息）

输出: 学园都市0313_split.json      ← 拆分后的世界书
```

---

## 3. 处理流程

```
输入A (世界书) + 输入B (分析结果)
    │
    ▼
1. load_world_book(A) → entries 字典
2. 加载 analysis JSON → 建立 uid → analysis_entry 映射
    │
    ▼
3. 遍历 entries，逐条目处理
    │
    ├─ analysis = uid_map[uid]
    ├─ 若 suggested_split == false → 原样保留
    ├─ 若 split_boundaries == null / [] → 原样保留
    ├─ 若 boundaries 内 is_dynamic 值全部相同 → 原样保留（防御）
    ├─ 否则执行拆分：
    │   ├─ 检测最外层 XML 标签
    │   ├─ 按 boundaries 切分 content 为段落
    │   ├─ 过滤空段落（仅空白行或仅含标题/空标签）
    │   ├─ 若过滤后静态+动态均有意义内容 → 创建2个子条目
    │   ├─ 若过滤后某侧为空 → 不拆分，原样保留
    │   └─ 子条目各自包裹最外层 XML 标签
    │
    └─ 累积处理结果
    │
    ▼
4. reassign_uids()（全量从 0 重排）
    │
    ▼
5. 移除 originalData（若存在，直接丢弃）
    │
    ▼
6. save_world_book → {原名}_split.json
```

---

## 4. 拆分算法详解

### 4.1 最外层 XML 标签检测

```python
def find_outermost_xml_tag(content: str) -> str | None:
    """
    匹配 <中文或英文标签>...</同一标签名> 的最外层标签对。

    规则：
    - 标签名含中文或英文字母+数字
    - 成对闭合（栈匹配）
    - 取在 content 中跨度最大、位于最外层的标签

    示例:
      "<角色>A<子>B</子>C</角色>" → "角色"
      "## 标题\n<事件>...</事件>" → "事件"
      "无标签文本" → None
    """
```

### 4.2 段落切分与过滤

```python
lines = entry["content"].splitlines(keepends=True)

static_segments: list[str] = []
dynamic_segments: list[str] = []

for b in analysis["split_boundaries"]:
    segment_lines = lines[b["start_line"] : b["end_line"] + 1]
    segment = "".join(segment_lines)

    # 过滤：丢弃仅空白或仅含标题/空标签的段落
    if _is_empty_or_heading_only(segment):
        continue

    if b["is_dynamic"]:
        dynamic_segments.append(segment)
    else:
        static_segments.append(segment)
```

#### `_is_empty_or_heading_only(segment)` 判定

```python
def _is_empty_or_heading_only(segment: str) -> bool:
    """
    段落去除所有 XML 标签、Markdown 标题（# / ##）、空白符后为空 → True
    即：仅含空白、标题行、空 XML 标签 → 丢弃
    """
    clean = re.sub(r"<[^>]+>", "", segment)                       # 去 XML 标签
    clean = re.sub(r"^#{1,2}\s.*$", "", clean, flags=re.MULTILINE) # 去标题行
    clean = clean.strip()
    return clean == ""
```

#### 空段落过滤详情

| 过滤场景 | 示例 | 是否丢弃 |
|---|---|---|
| 仅空白行 | `"\n\n"` | 丢弃 |
| 仅标题无正文 | `"## 人物介绍\n"` | 丢弃 |
| 仅 XML 标签无内容 | `"<角色></角色>"` | 丢弃 |
| 标题 + XML 标签无正文 | `"## 设定\n<标签></标签>"` | 丢弃 |
| 有实质文本 | `"## 人物介绍\n学园都市排名第三"` | 保留 |
| 含变量/宏 | `"<% getvar('x') %>"` | 保留 |

### 4.3 拆分判定

```python
# 防御1: 全静态或全动态 → 不拆分
is_dynamic_values = {b["is_dynamic"] for b in analysis["split_boundaries"]}
if len(is_dynamic_values) <= 1:
    new_entries.append(entry)  # 原样保留
    continue

# 防御2: 过滤后某侧为空 → 不拆分
if not static_segments or not dynamic_segments:
    new_entries.append(entry)  # 原样保留
    continue
```

### 4.4 组装子条目 content

```python
outermost_tag = find_outermost_xml_tag(entry["content"])

def wrap_content(segments: list[str], tag: str | None) -> str:
    body = "\n\n".join(segments)
    if tag:
        return f"<{tag}>\n{body}\n</{tag}>"
    return body

static_content = wrap_content(static_segments, outermost_tag)
dynamic_content = wrap_content(dynamic_segments, outermost_tag)
```

### 4.5 创建新条目

```python
# 继承原条目字段（排除 uid / content / comment）
base = {k: v for k, v in entry.items() if k not in ("uid", "content", "comment")}

static_entry = {
    **base,
    "uid": None,  # 待 reassign_uids 分配
    "content": static_content,
    "comment": f"{entry.get('comment', '')} [split-static]".strip(),
}

dynamic_entry = {
    **base,
    "uid": None,
    "content": dynamic_content,
    "comment": f"{entry.get('comment', '')} [split-dynamic]".strip(),
}

new_entries.extend([static_entry, dynamic_entry])
```

## 5. 字段继承策略

新创建的子条目从原条目继承以下字段：

| 字段 | 策略 |
|---|---|
| `key` / `keysecondary` | 继承（后续脚本4会清空静态 key） |
| `constant` / `disable` | 继承（后续脚本3会修改） |
| `selective` / `selectiveLogic` | 继承 |
| `order` / `position` / `depth` / `role` | 继承（后续脚本3会重新分配/修改） |
| `probability` / `useProbability` | 继承（后续脚本3会修改静态条目） |
| `sticky` / `cooldown` / `delay` | 继承（后续脚本3会修改） |
| `scanDepth` / `caseSensitive` / `matchWholeWords` / `useGroupScoring` | 继承（nullable 字段保持原值） |
| `triggers` | 继承 |
| `group` / `groupWeight` / `groupOverride` | 继承 |
| `characterFilter.*` | 继承 |
| `matchPersonaDescription` / `matchCharacterDescription` 等 | 继承 |
| `excludeRecursion` / `preventRecursion` / `delayUntilRecursion` | 继承 |
| `addMemo` / `outletName` / `automationId` / `ignoreBudget` / `vectorized` | 继承 |
| `uid` | **不继承** — 拆分后全量 reassign_uids() |
| `content` | **不继承** — 由拆分算法重新生成 |
| `comment` | 原值 + ` [split-static]` 或 ` [split-dynamic]` 后缀 |
| `displayIndex` | 继承（在 reassign_uids 后与 uid 相等） |

## 6. 不可拆分的情况（原样保留）

| 条件 | 说明 |
|---|---|
| `suggested_split == false` | 用户手动关闭 OR 脚本 1 判定不可拆分 |
| `split_boundaries == null` 或 `[]` | 脚本 1 未检测到有效边界 |
| 所有 boundary 的 `is_dynamic` 值相同 | 全静态或全动态，防御性跳过 |
| 过滤后静态片段为空 | 无有意义的静态段落 |
| 过滤后动态片段为空 | 无有意义的动态段落 |
| 纯静态条目 (`is_static=true`) | 脚本 1 判定 |
| 纯动态条目 (`is_mixed=false, is_static=false`) | 脚本 1 判定 |

---

## 7. originalData 处理

拆分后的世界书**直接丢弃** `originalData` 字段。

> 原因：拆分打破了 Spec v2 数组格式与内部字典格式的对应关系。再次导入酒馆并链接到角色卡时，酒馆会自动从 `entries` 重新生成 `originalData`。

## 8. 语义等价性

### 8.1 拆分前 vs 拆分后

```text
原条目:
   <御坂美琴>
   ## 人物介绍
   学园都市排名第三的Level 5超能力者...
   ## 关系状态
   当前态度: <%- getvar('relation') %>
   </御坂美琴>

拆分后:
   静态子条目:
     <御坂美琴>
     ## 人物介绍
     学园都市排名第三的Level 5超能力者...
     </御坂美琴>

   动态子条目:
     <御坂美琴>
     ## 关系状态
     当前态度: <%- getvar('relation') %>
     </御坂美琴>

→ 所有原始信息保留，各自独立包裹标签，语义无变化
```

### 8.2 验证规则

```
1. 静态 content + 动态 content 中包含原始条目的所有非空白文本内容
2. XML 标签正确闭合，无交叉
3. 不含孤立的 <% } %> / <% else { %> 残片（已在脚本1边界检测中排除）
```

---

## 9. 幂等性

| 输入场景 | 行为 |
|---|---|
| `_split.json` 再次输入脚本 1→2 | 静态子条目 → `is_static=true, suggested_split=false` → 保留 |
| | 动态子条目 → `is_mixed=false` → 保留 |
| | 不会再次追加 `[split-static]` 后缀 |
| 手动修改 comment 移除 `[split-*]` | 不影响判定逻辑（判定仅看 analysis 的 `is_dynamic`） |
