# 脚本 4 设计文档：静态条目合并器 `merge_entries.py`

---

## 1. 输入参数

```bash
python merge_entries.py --input <重排后世界书.json> [--output <输出路径.json>]
```

| 参数 | 必需 | 默认值 | 说明 |
|---|---|---|---|
| `--input` | 是 | — | 脚本 3 输出的重排后世界书 JSON |
| `--output` | 否 | `{输入文件名}_merged.json` | 合并后世界书输出路径 |

---

## 2. 处理流程

```
输入: 重排后世界书.json
    │
    ▼
1. load_world_book(input) → entries 列表
    │
    ▼
2. 分类：static_entries / non_mergeable_entries
    │  └─ 复用 detect_markers() + comment 标记判定
    │     ├─ 含动态标记 → non_mergeable
    │     ├─ comment 含 "[boundary-copy-" → non_mergeable
    │     ├─ comment 含 "[supplement-" → non_mergeable
    │     ├─ position=7 (outlet) → non_mergeable
    │     └─ 其余 → 可合并静态
    ▼
3. 可合并静态条目按 (position, constant, depth) 分组
    │
    ├─ 组内 >= 2 → 合并为1条
    └─ 组内 == 1 → 原样保留
    │
    ▼
4. 动态条目 + 边界副本 + 补充包裹 + outlet 条目全部原样保留，不参与合并
    │
    ▼
5. 合并后条目与 unmergeable 按 order 合并排序，重分配 order (0,1,2...)
    │
    ▼
6. 重分配 uid

---

## 3. 分类函数 `is_mergeable_static()`

```python
def is_mergeable_static(content: str, comment: str) -> bool:
    """
    判定条目是否为"可合并的静态"。

    排除：
    - 含动态标记的条目（detect_markers 非空）
    - 边界副本：comment 含 "[boundary-copy-"
    - 补充包裹：comment 含 "[supplement-"
    """
    if "[boundary-copy-" in comment:
        return False
    if "[supplement-" in comment:
        return False
    if not determine_static(content):
        return False
    return True
```

> 边界副本（`<暗部>`）和补充包裹（`<补充内容>`）的 content 不含任何动态标记，`determine_static()` 会误判为静态。但它们都共用 `pos=4, constant=false, depth=0`，不排除的话会将所有边界副本和补充包裹错误合并为同一条，破坏包裹结构。

## 4. 合并算法

### 4.1 分组 → 合并

```python
def merge_static_entries(entries: list[dict]) -> tuple[list[dict], list[dict]]:
    """
    返回 (merged_static, non_mergeable)
    """
    mergeable = []
    non_mergeable = []
    for e in entries:
        if is_mergeable_static(e["content"], e.get("comment", "")):
            mergeable.append(e)
        else:
            non_mergeable.append(e)

    groups = defaultdict(list)
    for e in statics:
        groups[(e["position"], e["constant"], e["depth"])].append(e)

    merged = []
    for group in groups.values():
        group.sort(key=lambda e: e["order"])
        if len(group) == 1:
            merged.append(group[0])  # 单条不合并
        else:
            merged.append(_merge_one_group(group))

    return merged, dynamics
```

### 3.2 单组合并

```python
def _merge_one_group(group: list[dict]) -> dict:
    # content 按 order 升序拼接
    contents = [e["content"] for e in group]
    merged_content = "\n\n".join(contents)

    # order: 组内最小值
    min_order = group[0]["order"]
    max_order = group[-1]["order"]

    # comment: 使用 order 范围（uid 合并后会变，order 在脚本3后固定）
    comment = f"合并:{min_order}-{max_order}"

    # 继承组内首条作为基底
    base = group[0]

    return {
        # === 分组字段：继承 ===
        "position": base["position"],
        "constant": base["constant"],
        "depth": base["depth"],

        # === 合并字段：覆盖 ===
        "content": merged_content,
        "order": min_order,
        "comment": comment,
        "key": [],
        "keysecondary": [],

        # === 功能字段：继承脚本3设定的值 ===
        "role": base["role"],
        "useProbability": False,

        # === 其余字段：取默认值 ===
        "disable": False,
        "selective": True,
        "selectiveLogic": 0,
        "probability": 100,
        "sticky": 0,
        "cooldown": 0,
        "delay": 0,
        "scanDepth": None,
        "caseSensitive": None,
        "matchWholeWords": None,
        "useGroupScoring": None,
        "matchPersonaDescription": False,
        "matchCharacterDescription": False,
        "matchCharacterPersonality": False,
        "matchCharacterDepthPrompt": False,
        "matchScenario": False,
        "matchCreatorNotes": False,
        "excludeRecursion": False,
        "preventRecursion": False,
        "delayUntilRecursion": False,
        "ignoreBudget": False,
        "group": "",
        "groupOverride": False,
        "groupWeight": 100,
        "outletName": "",
        "automationId": "",
        "vectorized": False,
        "addMemo": False,
        "triggers": [],
        "characterFilter": {"names": [], "tags": [], "isExclude": False},
        "displayIndex": None,

        # === uid：待 reassign_uids 分配 ===
        "uid": None,
    }
```

## 4. comment 使用 order 而非 uid 的原因

合并后 `reassign_uids()` 会重新分配 uid，合并前的 uid 在输出中不再存在。

```
合并前（脚本3输出）:
  uid=3  content="A" order=0
  uid=7  content="B" order=1
  uid=12 content="C" order=2

合并后:
  uid 重分配 → uid=0, 不存在原来的 3/7/12

若用 uid:  "合并:3-12"   → 无法在输出中追溯到任何条目
若用 order: "合并:0-2"   → 可追溯到脚本3排序后的位次
```

## 5. 字段策略表

| 字段 | 来源 | 理由 |
|---|---|---|
| `position` / `constant` / `depth` | 分组 key，继承 | 合并前提条件 |
| `content` | `\n\n` 拼接 | 合并核心操作 |
| `order` | 组内最小值 | 保持最靠前 |
| `comment` | `"合并:{min_order}-{max_order}"` | 可追溯到脚本3排序位次 |
| `key` / `keysecondary` | `[]` | 蓝灯条目无需关键词匹配 |
| `role` | 继承组内值 | 脚本3设定了 atDepth 注入角色（User=1） |
| `useProbability` | `false` | 保证合并条目必定激活 |
| 其余字段 | 默认值 | key 清空后功能退化，无实际影响；参见下文分析 |

### 其余字段取默认值的原因

| 字段组 | 默认值 | 取默认值的理由 |
|---|---|---|
| `probability` / `cooldown` / `sticky` / `delay` | `100` / `0` / `0` / `0` | 与脚本3设定值一致，无冲突 |
| `selective` / `selectiveLogic` | `true` / `0` | key 已清空，选择性逻辑不生效 |
| `scanDepth` / `caseSensitive` / `matchWholeWords` | `null` | key 已清空，匹配参数无意义 |
| `matchPersonaDescription` 等匹配源 | `false` | 蓝灯条目常驻激活，无需关键词匹配源 |
| `group` / `groupWeight` / `groupOverride` | `""` / `100` / `false` | 合并后组内仅1条，包含组机制无意义 |
| `triggers` | `[]` | constant=true 已保证常驻激活 |
| `excludeRecursion` / `preventRecursion` | `false` | 蓝灯条目不参与递归扫描 |
| `outletName` / `automationId` | `""` | 合并对象是 pos4 蓝灯，不涉及 outlet 或自动化 |

## 7. 动态条目 / 边界副本 / 补充包裹处理

- **动态条目**：`determine_static(content)` 返回 False → 归入 `non_mergeable` → 不合并
- **边界副本**：`is_mergeable_static()` 通过 comment 排除 → 不合并
- **补充包裹**：`is_mergeable_static()` 通过 comment 排除 → 不合并
- 所有 `non_mergeable` 条目保持原样
- 但参与最终 `reassign_uids()`，与合并后的静态条目统一排序

## 9. 幂等性

| 输入场景 | 行为 |
|---|---|
| `_merged.json` 再次作为输入 | 同组静态条目已合并为1条 → 组内1条 → 不合并 |
| | 动态条目保持不变 |
| | comment `"合并:..."` 不再追加嵌套 |
| | 两次输出一致 ✓ |
