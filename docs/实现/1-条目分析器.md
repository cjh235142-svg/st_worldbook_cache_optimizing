## 3. 处理流程

```
输入世界书.json
    │
    ▼
1. （原始文件备份由 run_pipeline.py 在入口统一执行）
    │
    ▼
2. load_world_book(input) → entries 字典
    │
    ▼
3. 遍历 entries，逐条目分析
    │
    ├─ 3a. detect_markers(content) → 动态标记列表
    ├─ 3b. parse_content_blocks(content) → text/ejs 块序列
    ├─ 3c. find_ejs_compound_ranges(blocks) → EJS 复合块行范围
    ├─ 3d. 三态判定（考虑 EJS 复合块覆盖率）
    ├─ 3e. 若 is_mixed → 检测拆分边界（排除复合块内边界）
    └─ 3f. 累积统计计数
    │
    ▼
4. 组装输出 JSON
    │
    ▼
5. 写入 {原名}_analysis.json
```

## 4. EJS 复合块识别（核心增强）

### 4.1 动机

EJS 的 `if/else/for/while` 结构跨越多个 `<% %>` → `%>` 交替块：

```text
<% if (cond) { %>        ← EJS块1, brace深度变为+1
  文本输出A
<% } else { %>           ← EJS块2, brace净变化=0 (深度仍为1)
  文本输出B
<% } %>                  ← EJS块3, brace深度-1=0（闭合）
```

如果机械按 `#` 标题或 `##` 标题切分，会割裂 EJS 结构，产生孤立的 `<% } %>` 语义残片。

### 4.2 数据结构

```python
from dataclasses import dataclass

@dataclass
class ContentBlock:
    type: str          # "text" | "ejs"
    content: str       # 块内容（不含 <% %> 标记）
    start_line: int    # 在原始 content 中的起始行号（0-based）
    end_line: int      # 在原始 content 中的结束行号（含）
    brace_delta: int   # 该 EJS 块的净 brace 变化（{数 - }数），text 块为 0
```

### 4.3 `parse_content_blocks(content) -> list[ContentBlock]`

将 content 按 `<%...%>` 边界切分为 text/ejs 交替序列。

**步骤**：
1. 按行分割 content
2. 扫描所有包含 `<%` 和 `%>` 的行，确定 EJS 块的起始/结束行
3. 跨行 EJS 块合并为单一块
4. 计算每个 EJS 块的 `brace_delta`（调用 `count_net_braces`）

**正则**：
- EJS 非输出块开始：`<%(?![-=])\s*`
- EJS 转义输出：`<%=`
- EJS 非转义输出：`<%-`

### 4.4 `count_net_braces(ejs_code: str) -> int`

统计 EJS 代码块中 `{` 和 `}` 的差值。

```python
def count_net_braces(code: str) -> int:
    """
    返回 { 数 - } 数。
    排除字符串字面量和模板字符串中的括号。
    """
    clean = _strip_string_literals(code)
    return clean.count("{") - clean.count("}")
```

### 4.5 `find_ejs_compound_ranges(content) -> list[tuple[int, int]]`

```python
def find_ejs_compound_ranges(content: str) -> list[tuple[int, int]]:
    """
    扫描所有 EJS 块，追踪 brace 深度。
    从深度 > 0 开始，到深度 == 0 结束，形成一个复合块。
    返回 [(start_line, end_line), ...]
    """
    blocks = parse_content_blocks(content)
    ranges = []
    depth = 0
    start = None

    for block in blocks:
        if block.type == "ejs":
            depth += block.brace_delta
            if depth > 0 and start is None:
                start = block.start_line
            if depth == 0 and start is not None:
                ranges.append((start, block.end_line))
                start = None

    return ranges
```

**示例**：

```
输入:
  line 0: # 状态总览
  line 1: <% if (cond) { %>
  line 2: # 阶段一事件
  line 3: 事件A
  line 4: <% } else { %>
  line 5: # 阶段二事件
  line 6: 事件B
  line 7: <% } %>

输出: [(1, 7)]
```

### 4.6 EJS 复合块对 `is_mixed` 的影响

```python
def classify(content: str) -> tuple[bool, bool]:
    """
    返回 (is_static, is_mixed)
    """
    markers = detect_markers(content)
    if not markers:
        return (True, False)  # 纯静态

    # 移除所有 EJS 块和宏，检查残留
    clean = _strip_ejs_and_macros(content).strip()
    if not clean:
        return (False, False)  # 纯动态

    # 有静态残留，检查是否被 EJS 复合块完全覆盖
    compound_ranges = find_ejs_compound_ranges(content)
    if compound_ranges and _content_fully_covered(content, compound_ranges):
        return (False, False)  # 被复合块覆盖 → 不可拆分

    return (False, True)  # 混合可拆分
```

## 6. 拆分边界检测

### 6.1 检测流程

```
1. find_ejs_compound_ranges(content) → 不可拆分行范围集合
2. 找到所有候选边界：Markdown 标题行（`^#{1,2}\s`）作为段落切分起点
3. 过滤：若标题段落的 start 或 end 行号落在任一 EJS 复合块区间内：
   - 单行段落且全部在复合块内 → 保留，按内容判定 is_dynamic
   - 跨边界段落（部分在复合块内、部分在外）→ 保留，强制标记 is_dynamic=True（保护 EJS 完整性）
4. 递归闭标签二次切分：对已按 heading 切好的段落，在 `</tagname>` 独立行处再次切分
5. 分配 wrap_tag：`</tag>` 之后的段落 set wrap_tag=None（不包裹 tag），之前的段落 set wrap_tag=tag_name
6. 每段落判定 is_dynamic：
   - 段落文本匹配任何动态标记 → true
   - 否则 → false
```

### 6.2 过滤示意

```text
场景:
  line 0: # 状态总览          ← 候选边界, 复合块外 → 保留 (static)
  line 1: <% if (cond) { %>   ← EJS复合块开始: lines 1-7
  line 2: # 阶段一事件         ← 候选边界, 跨越复合块→ 保留, 强制标记 is_dynamic=True
  line 3: 事件A的描述
  line 4: <% } else { %>
  line 5: # 阶段二事件         ← 候选边界, 跨越复合块→ 保留, 强制标记 is_dynamic=True
  line 6: 事件B的描述
  line 7: <% } %>             ← EJS复合块闭合
  line 8: # 另一个标题         ← 候选边界, 复合块外 → 保留 (按内容判定)

最终拆分点: line 0, line 2, line 5, line 8
段落1: lines 0-1 (含 <% if{_%> 开头 → is_dynamic=true)
段落2: lines 2-4 (含 EJS 中间部分 → is_dynamic=true)
段落3: lines 5-7 (含 <% } %> 闭合 → is_dynamic=true)
段落4: lines 8+  (复合块外 → 按内容判定)
```

## 7. 特殊分支处理

| 场景 | 处理方式 |
|---|---|
| **空 content** (`""`) | `is_static=true, is_mixed=false`, markers=[], boundaries=null |
| **仅有空白** (`"\n \t"`) | 同空 content |
| **outlet 条目** (`position=7`) | `is_mixed=false, suggested_split=false`，不参与拆分、重排序、合并 |
| **插件标记条目** (`[GENERATE:BEFORE]`/`[InitialVariables]`/`@@preprocessing` 等) | `is_mixed=false, suggested_split=false`，不参与拆分 |
| **disabled 条目** (`disable=true`) | 正常分析，不跳过（disable 在后续脚本中处理） |
| **静态宏**（`{{char}}`、`{{user}}` 等） | 不在检测列表 → `is_static=true` |
| **纯 EJS 全覆盖** | `is_mixed=false`（clean 后无残留） |
| **EJS 复合块全覆盖** | `is_mixed=false, suggested_split=false` |
| **EJS 复合块未闭合** | 标记为不可拆分（`suggested_split=false`），保留用户手动处理空间 |
| **嵌套 EJS 复合块** | 按最外层 brace 归零判定范围；外层范围的行号已覆盖内层嵌套区域，内层不单独列出 |
| **`for` / `while` 块** | brace 跟踪统一处理，不区分关键字类型 |
| **混合不可拆分** | `is_mixed=true, suggested_split=false`（如无有效拆分边界或残留全在复合块内） |
| **entries 为空字典 `{}`** | 不报错，输出 entries=[], summary 全 0 |
| **JSON 解析失败** | 报错退出 |
| **文件不存在** | 报错退出 |
