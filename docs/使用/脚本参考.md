# 脚本 CLI 参考

## 管线主控 `run_pipeline.py`

一键串联四步管线。

```bash
python -m scripts.run_pipeline -i <世界书.json> [参数]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `-i`, `--input` | 是 | — | 原始世界书 JSON 路径 |
| `-d`, `--output-dir` | 否 | 输入文件所在目录 | 输出产物目录 |
| `-w`, `--wrapper-name` | 否 | `补充内容` | 补充包裹的内容名称 |

**使用示例：**

```bash
python -m scripts.run_pipeline -i 学园都市0313.json
python -m scripts.run_pipeline -i test.json -d out -w "可变设定"
```

## 脚本 1 `analyze_entries.py`

分析每条 entry 的内容：静态 / 动态 / 混合可拆分 / 混合不可拆分。

```bash
python -m scripts.analyze_entries -i <世界书.json> [-o <路径>]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `-i`, `--input` | 是 | — | 世界书 JSON 路径 |
| `-o`, `--output` | 否 | `{原名}_analysis.json` | 分析结果输出路径 |

**输出 JSON 关键字段：**

| 字段 | 说明 |
|------|------|
| `is_static` | 是否不含动态标记 |
| `is_mixed` | 是否混合（含静态+动态） |
| `suggested_split` | 是否建议拆分（outlet/插件条目强制 false） |
| `detected_markers` | 命中的动态标记列表 |
| `split_boundaries` | 建议的拆分边界（标题行范围） |

**使用示例：**

```bash
python -m scripts.analyze_entries -i 世界书.json
# → 世界书_analysis.json

python -m scripts.analyze_entries -i 世界书.json -o result/a.json
```

> 用户可编辑 `_analysis.json`，将 `suggested_split: true` 改为 `false` 来跳过特定条目的拆分。

## 脚本 2 `split_entries.py`

根据分析结果将混合条目拆分为静态子条目和动态子条目。

```bash
python -m scripts.split_entries -i <世界书.json> -a <分析结果.json> [-o <路径>]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `-i`, `--input` | 是 | — | 原始世界书 JSON（只读） |
| `-a`, `--analysis` | 是 | — | 脚本1输出的分析 JSON |
| `-o`, `--output` | 否 | `{世界书原名}_split.json` | 拆分后输出路径 |

**拆分行为：**
- 沿 Markdown 标题（`#`/`##`）边界切分
- 子条目各自包裹原最外层 XML 标签
- 子条目 comment 附加 `[split-static]` 或 `[split-dynamic]`
- 跳过条件：outlet 条目、插件标记条目、EJS 全覆盖条目

**使用示例：**

```bash
python -m scripts.split_entries \
    -i 学园都市0313.json \
    -a 学园都市0313_analysis.json
```

## 脚本 3 `reorder_entries.py`

重排序条目 + 字段重映射 + 边界副本创建 + 补充包裹注入。

```bash
python -m scripts.reorder_entries -i <拆分后世界书.json> [-o <路径>] [-w <名称>]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `-i`, `--input` | 是 | — | 脚本2输出的拆分后世界书 |
| `-o`, `--output` | 否 | `{原名}_reordered.json` | 输出路径 |
| `-w`, `--wrapper-name` | 否 | `补充内容` | 补充包裹的内容名称 |

**核心行为：**

| 条目类型 | 字段变化 |
|----------|---------|
| 静态 pos4 depth<10 | `constant=true, depth=0, role=1` |
| 静态 pos4 depth≥10 | `constant=true, depth=9999, role=1` |
| 静态 pos≠4 | `constant=true`（pos/depth/role 不变） |
| 动态 | `position=4, depth=0, role=1` |
| outlet (pos=7) | **不修改** |
| 边界副本/补充包裹 | `constant=false, key=["/.*/"], pos=4, depth=0, useProbability=false, probability=100, cooldown=0, sticky=0, delay=0` |

**排序键（6键升序）：**

```
(category, sub_order, stability, position, depth_key, _original_order)
```

- `category`: 静态=0，动态/副本/包裹=1
- `sub_order`: 包裹开=0，内容=1，包裹闭=2
- `stability`: pos4深度≥10=0(核心), pos≠4=1(过渡), pos4深度<10=2(变化)

**使用示例：**

```bash
python -m scripts.reorder_entries -i 学园都市0313_split.json
python -m scripts.reorder_entries -i in.json -w "可变内容"
```

## 脚本 4 `merge_entries.py`

合并同 `(position, constant, depth)` 的静态条目。

```bash
python -m scripts.merge_entries -i <重排后世界书.json> [-o <路径>]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `-i`, `--input` | 是 | — | 脚本3输出的重排后世界书 |
| `-o`, `--output` | 否 | `{原名}_merged.json` | 最终输出路径 |

**合并规则：**
- 同组静态条目 content 用 `\n\n` 拼接
- key/keysecondary 清空（蓝灯无需关键词）
- comment 改为 `"合并:{min}-{max}"`
- 动态条目不参与合并
- 合并后重新分配 order 和 uid

**使用示例：**

```bash
python -m scripts.merge_entries -i 学园都市0313_reordered.json
```

## 验证工具 `tool_validate.py`

检查世界书的 XML 配对、EJS 完整性、宏闭合、缓存规则等。

```bash
python -m scripts.tools.tool_validate -i <世界书.json> [--output-errors <路径>] [--strict]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `-i`, `--input` | 是 | — | 世界书 JSON 路径 |
| `--output-errors` | 否 | — | 错误列表输出为 JSON（机器可读） |
| `--strict` | 否 | false | 额外检查缓存优化建议规则 |
| `--pipeline-check` | 否 | false | 对单文件执行产物完整性检查 |
| `--pipeline-dir` | 否 | — | 整套产物所在目录 |
| `--pipeline-name` | 否 | — | 产物文件名前缀 |

**检查项：**

| 级别 | 类别 | 说明 |
|------|------|------|
| 🔴 错误 | XML 配对 | 条内标签未闭合/跨条目标签未闭合 |
| 🔴 错误 | EJS 完整性 | `<%` 无 `%>` 或 brace 不平衡 |
| 🔴 错误 | 结构合法性 | uid 与 key 不一致、content 缺失 |
| 🟡 警告 | 宏完整性 | `{{` 与 `}}` 数量不匹配 |
| 🟡 警告 | 缓存规则 (--strict) | 蓝灯 key 非空、静态 probability<100 等 |

**使用示例：**

```bash
# 常规检查
python -m scripts.tools.tool_validate -i 学园都市0313_merged.json --strict

# 单文件管线完整性检查
python -m scripts.tools.tool_validate -i 学园都市0313_merged.json --pipeline-check

# 整套产物链检查
python -m scripts.tools.tool_validate \
    --pipeline-dir /tmp/opencode/test_output/ \
    --pipeline-name 学园都市0313
```

> 注意：大多数 XML "错误" 是 `<user>`（宏占位符）或 `<br>`（HTML 自闭合标签）的误报，可安全忽略。

## 查询工具 `tool_query.py`

跨产物查询、过滤、验证。按类型、order 范围、文本搜索筛选，支持 CSV 输出和产物完整性验证。

```bash
python -m scripts.tools.tool_query -i <产物.json> \
    [--filter static|dynamic|boundary|supplement|merged] \
    [--order-range N M] [--search <文本>] \
    [--csv] [--summary-only] [--verify]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `-i`, `--input` | 是 | — | 管线产物 JSON（任意步骤） |
| `--filter` | 否 | — | 类型筛选：`static` / `dynamic` / `boundary` / `supplement` / `merged` |
| `--order-range` | 否 | — | order 范围，两个数字如 `0 9` |
| `--search` | 否 | — | 模糊匹配 comment 或 content |
| `--csv` | 否 | — | CSV 输出（可导入 Excel） |
| `--summary-only` | 否 | — | 仅输出统计数字 |
| `--verify` | 否 | — | 产物完整性验证 |

**使用示例：**

```bash
python -m scripts.tools.tool_query -i 学园都市0313_merged.json --summary-only
python -m scripts.tools.tool_query -i 学园都市0313_merged.json --filter boundary
python -m scripts.tools.tool_query -i 学园都市0313_merged.json --filter merged --csv
python -m scripts.tools.tool_query -i 学园都市0313_merged.json --verify
python -m scripts.tools.tool_query -i 学园都市0313_split.json --search "御坂美琴"
```

## 完整列表工具 `tool_list_full.py`

按排序列出条目 + 完整内容。支持按 UID 精确定位或按文本搜索。

```bash
python -m scripts.tools.tool_list_full -i <世界书.json> [-o <路径>]
    [--max-len 500] [--no-truncate] [--filter static|dynamic]
    [-u <UID>] [-s <文本>]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `-i`, `--input` | 是 | — | 世界书 JSON 路径 |
| `-o`, `--output` | 否 | — | 输出到文件（默认 stdout） |
| `--max-len` | 否 | 500 | 每条 content 最大字符数，截断时追加 `...(truncated)` |
| `--no-truncate` | 否 | — | 显示完整 content |
| `--filter` | 否 | — | 过滤：`static` / `dynamic` / 省略=全部 |
| `-u`, `--uid` | 否 | — | 按 UID 精确定位单一条目，只输出该条目。与 `-s` 同时存在时优先 |
| `-s`, `--search` | 否 | — | 模糊匹配 `comment` 或 `content` 包含该文本的条目，列出所有匹配项 |

**使用示例：**

```bash
python -m scripts.tools.tool_list_full -i 学园都市0313_merged.json
python -m scripts.tools.tool_list_full -i test.json --max-len 200 --filter dynamic
python -m scripts.tools.tool_list_full -i 学园都市0313_merged.json -u 5
python -m scripts.tools.tool_list_full -i 学园都市0313_merged.json -s "御坂美琴" --no-truncate
```

| 参数 | 必需 | 默认值 | 说明 |

| `-C`, `--no-comments` | 否 | — | 仅输出纯内容（HTML 注释），可直接复制测试缓存命中 |

**使用示例：**

## 对比工具 `tool_diff.py`

按 6 键排序对比，追踪原始条目的去向（拆分/合并/保持）和优化后条目的来源。支持按 UID 或文本筛选。

```bash
python -m scripts.tools.tool_diff --original <原始.json> --optimized <优化后.json>
    [-f unified|json] [--output <路径>]
    [--from-uid <UID>] [--from-search <文本>]
    [--to-uid <UID>] [--to-search <文本>]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--original` | 是 | — | 原始世界书 JSON |
| `--optimized` | 是 | — | 管线优化后的世界书 JSON（通常为 merged 产物） |
| `-f`, `--format` | 否 | unified | 输出格式：`unified`（人类可读）或 `json`（机器可读） |
| `--output` | 否 | — | 输出到文件（默认 stdout） |
| `--from-uid` | 否 | — | 向前追踪：按原始条目 UID 过滤，可重复（如 `--from-uid 1 --from-uid 29`），多个取交集 |
| `--from-search` | 否 | — | 向前追踪：按文本过滤原始条目（匹配 comment 或 content），与 `--from-uid` 取交集 |
| `--to-uid` | 否 | — | 反向追踪：按优化后条目 UID 过滤，可重复，多个取交集 |
| `--to-search` | 否 | — | 反向追踪：按文本过滤优化后条目，与 `--to-uid` 取交集 |

**使用示例：**

```bash
# 完整 diff
python -m scripts.tools.tool_diff --original 学园都市0313.json --optimized 学园都市0313_merged.json

# 向前追踪：指定原始条目 UID
python -m scripts.tools.tool_diff --original 学园都市0313.json --optimized 学园都市0313_merged.json \
    --from-uid 1 --from-uid 29

# 向前追踪：按文本搜索
python -m scripts.tools.tool_diff --original 学园都市0313.json --optimized 学园都市0313_merged.json \
    --from-search "御坂美琴"

# 反向追踪：查看合并组来源
python -m scripts.tools.tool_diff --original 学园都市0313.json --optimized 学园都市0313_merged.json \
    --to-uid 0

# JSON 格式
python -m scripts.tools.tool_diff --original in.json --optimized out.json -f json
```
