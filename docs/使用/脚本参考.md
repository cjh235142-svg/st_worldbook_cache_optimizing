# 脚本 CLI 参考

## 管线主控 `run_pipeline.py`

一键串联四步管线。

```bash
python -m scripts.run_pipeline --input <世界书.json> [参数]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--input` | 是 | — | 原始世界书 JSON 路径 |
| `--output-dir` | 否 | 输入文件所在目录 | 输出产物目录 |
| `--wrapper-name` | 否 | `补充内容` | 补充包裹的内容名称 |

**使用示例：**

```bash
python -m scripts.run_pipeline --input 学园都市0313.json
python -m scripts.run_pipeline --input test.json --output-dir out --wrapper-name "可变设定"
```

## 脚本 1 `analyze_entries.py`

分析每条 entry 的内容：静态 / 动态 / 混合可拆分 / 混合不可拆分。

```bash
python -m scripts.analyze_entries --input <世界书.json> [--output <路径>]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--input` | 是 | — | 世界书 JSON 路径 |
| `--output` | 否 | `{原名}_analysis.json` | 分析结果输出路径 |

**输出 JSON 关键字段：**

| 字段 | 说明 |
|------|------|
| `is_static` | 是否不含动态标记 |
| `is_mixed` | 是否混合（含静态+动态） |
| `suggested_split` | 是否建议拆分（outlet/插件条目强制 false） |
| `detected_markers` | 命中的动态标记列表 |
| `split_boundaries` | 建议的拆分边界（标题行范围） |

**使用示例：**

```bash
python -m scripts.analyze_entries --input 世界书.json
# → 世界书_analysis.json

python -m scripts.analyze_entries --input 世界书.json --output result/a.json
```

> 用户可编辑 `_analysis.json`，将 `suggested_split: true` 改为 `false` 来跳过特定条目的拆分。

## 脚本 2 `split_entries.py`

根据分析结果将混合条目拆分为静态子条目和动态子条目。

```bash
python -m scripts.split_entries --world-book <世界书.json> --analysis <分析结果.json> [--output <路径>]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--world-book` | 是 | — | 原始世界书 JSON（只读） |
| `--analysis` | 是 | — | 脚本1输出的分析 JSON |
| `--output` | 否 | `{世界书原名}_split.json` | 拆分后输出路径 |

**拆分行为：**
- 沿 Markdown 标题（`#`/`##`）边界切分
- 子条目各自包裹原最外层 XML 标签
- 子条目 comment 附加 `[split-static]` 或 `[split-dynamic]`
- 跳过条件：outlet 条目、插件标记条目、EJS 全覆盖条目

**使用示例：**

```bash
python -m scripts.split_entries \
    --world-book 学园都市0313.json \
    --analysis 学园都市0313_analysis.json
```

## 脚本 3 `reorder_entries.py`

重排序条目 + 字段重映射 + 边界副本创建 + 补充包裹注入。

```bash
python -m scripts.reorder_entries --input <拆分后世界书.json> [--output <路径>] [--wrapper-name <名称>]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--input` | 是 | — | 脚本2输出的拆分后世界书 |
| `--output` | 否 | `{原名}_reordered.json` | 输出路径 |
| `--wrapper-name` | 否 | `补充内容` | 补充包裹的内容名称 |

**核心行为：**

| 条目类型 | 字段变化 |
|----------|---------|
| 静态 pos4 depth<10 | `constant=true, depth=0, role=1` |
| 静态 pos4 depth≥10 | `constant=true, depth=9999, role=1` |
| 静态 pos≠4 | `constant=true`（pos/depth/role 不变） |
| 动态 | `position=4, depth=0, role=1` |
| outlet (pos=7) | **不修改** |
| 边界副本/补充包裹 | `key=["/.*/"], pos=4, depth=0` |

**排序键（6键升序）：**

```
(category, sub_order, stability, position, depth_key, _original_order)
```

- `category`: 静态=0，动态/副本/包裹=1
- `sub_order`: 包裹开=0，内容=1，包裹闭=2
- `stability`: pos4深度≥10=0(核心), pos≠4=1(过渡), pos4深度<10=2(变化)

**使用示例：**

```bash
python -m scripts.reorder_entries --input 学园都市0313_split.json
python -m scripts.reorder_entries --input in.json --wrapper-name "可变内容"
```

## 脚本 4 `merge_entries.py`

合并同 `(position, constant, depth)` 的静态条目。

```bash
python -m scripts.merge_entries --input <重排后世界书.json> [--output <路径>]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--input` | 是 | — | 脚本3输出的重排后世界书 |
| `--output` | 否 | `{原名}_merged.json` | 最终输出路径 |

**合并规则：**
- 同组静态条目 content 用 `\n\n` 拼接
- key/keysecondary 清空（蓝灯无需关键词）
- comment 改为 `"合并:{min}-{max}"`
- 动态条目不参与合并
- 合并后重新分配 order 和 uid

**使用示例：**

```bash
python -m scripts.merge_entries --input 学园都市0313_reordered.json
```

## 验证工具 `tool_validate.py`

检查世界书的 XML 配对、EJS 完整性、宏闭合、缓存规则等。

```bash
python -m scripts.tools.tool_validate --input <世界书.json> [--output-errors <路径>] [--strict]
```

| 参数 | 必需 | 默认值 | 说明 |
|------|------|--------|------|
| `--input` | 是 | — | 世界书 JSON 路径 |
| `--output-errors` | 否 | — | 错误列表输出为 JSON（机器可读） |
| `--strict` | 否 | false | 额外检查缓存优化建议规则 |

**检查项：**

| 级别 | 类别 | 说明 |
|------|------|------|
| 🔴 错误 | XML 配对 | 条内标签未闭合/跨条目标签未闭合 |
| 🔴 错误 | EJS 完整性 | `<%` 无 `%>` 或 brace 不平衡 |
| 🔴 错误 | 结构合法性 | uid 与 key 不一致、content 缺失 |
| 🟡 警告 | 宏完整性 | `{{` 与 `}}` 数量不匹配 |
| 🟡 警告 | 缓存规则 (--strict) | 蓝灯 key 非空、静态 probability<100 等 |

**使用示例：**

```bash
python -m scripts.tools.tool_validate --input 学园都市0313_merged.json --strict
```

> 注意：大多数 XML "错误" 是 `<user>`（宏占位符）或 `<br>`（HTML 自闭合标签）的误报，可安全忽略。
